# Геопривязка карты

Цель: пользователь загружает картинку спортивной карты, отмечает несколько точек
на изображении и соответствующие точки на базовой карте. Система получает
преобразование из пикселей картинки в географические координаты.

## Первый этап

Используем affine-преобразование:

```text
lon = a*x + b*y + c
lat = d*x + e*y + f
```

Минимум нужны 3 контрольные точки, но в интерфейсе лучше требовать 4-6.
Лишние точки используются как least-squares система, а качество оценивается
остаточной ошибкой в метрах.

Это хорошо работает для сканов/скриншотов без сильной перспективы, когда карта
только сдвинута, повернута и масштабирована.

## UX потока

1. Загрузить изображение карты.
2. Открыть экран с двумя областями: слева изображение, справа базовая карта.
3. Пользователь ставит одну и ту же точку на обеих картах.
4. После 3 точек показываем превью наложения и ошибку.
5. После 4-6 точек сохраняем привязку.
6. Трек тренировки отображается поверх картинки через `pixel_to_geo` или обратное преобразование.

## Следующий уровень

Для спортивных карт часто понадобится более точная модель:

- affine как MVP;
- projective/homography для перспективных искажений;
- thin plate spline или локальная сетка для деформированных сканов;
- экспорт результата как GeoJSON world file или локальный tileset.

Начинаем с affine, потому что он простой, тестируемый и сразу позволит проверить
весь пользовательский сценарий.
